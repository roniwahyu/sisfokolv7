# A4. Analisis Kebutuhan Bisnis

---

## Latar Belakang

SMP Islam Terpadu masih mengandalkan proses manual berbasis kertas dan spreadsheet terpisah, yang mengakibatkan keterlambatan pelaporan, duplikasi data, dan kesulitan monitoring.

## Tabel Masalah Bisnis + Dampak

| No | Masalah Bisnis (As-Is) | Penyebab Utama | Dampak | Prioritas |
| --- | --- | --- | --- | --- |
| 1 | Data siswa, guru, dan kelas tersimpan di banyak file Excel | Tidak ada basis data terpusat | Duplikasi dan inkonsistensi data | Tinggi |
| 2 | Input nilai memakan waktu 1–2 minggu | Proses manual per mata pelajaran | Rapor terlambat dan guru lembur | Tinggi |
| 3 | Absensi harian direkap manual di buku | Tidak ada sistem presensi digital | Laporan keterlambatan tidak real-time | Tinggi |
| 4 | Informasi pembayaran SPP tidak transparan bagi orang tua | Pencatatan hanya di bendahara | Klaim pembayaran sering terjadi | Tinggi |
| 5 | Laporan untuk kepala sekolah dibuat manual setiap akhir semester | Pengumpulan data dari banyak sumber | Keputusan tidak berbasis data real-time | Sedang |
| 6 | Komunikasi sekolah–orang tua bergantung grup informal | Tidak ada saluran resmi | Informasi akademik sering tercecer | Sedang |
| 7 | Proses cetak rapor memerlukan banyak koreksi | Data tidak terintegrasi | Kredibilitas rapor menurun | Tinggi |

## Manfaat yang Diharapkan (To-Be)

| Area | Manfaat Kuantitatif | Manfaat Kualitatif |
| --- | --- | --- |
| Akurasi Data | Duplikasi data < 1% | Kepercayaan stakeholder meningkat |
| Efisiensi Waktu | Input nilai 70% lebih cepat | Guru fokus pada pengajaran |
| Transparansi | 100% orang tua bisa cek tagihan | Konflik pembayaran berkurang |
| Keputusan | Laporan tersedia real-time | Kepala sekolah lebih responsif |
| Komunikasi | Notifikasi otomatis | Orang tua lebih terlibat |

## Rekomendasi Solusi

Implementasi **Sistem Informasi Sekolah Berbasis Web** dengan modul terintegrasi, otentikasi peran, audit trail, dan dashboard laporan.
